# hyper98.trade

A Windows 98–themed frontend for Hyperliquid.

## The bet

Hyperliquid's builder codes let any frontend earn up to 0.1% on every
trade routed through it. ~40% of Hyperliquid daily active users now
trade through third-party frontends. Phantom earns $100k/day doing
this. PvP.trade has earned $7M+ all-time. The game is on.

Every competing frontend looks the same: dark-mode Bloomberg terminal
or clean crypto minimal. hyper98 commits fully to a different angle —
a real Windows 98 desktop environment. Draggable windows, taskbar,
Start menu, pixel icons, MS Sans Serif. Not a skin. A full OS.

## Target user

Crypto-native perps traders who appreciate aesthetic commitment.
Not trying to be the cheapest or onboard non-crypto users. Trying to
be the frontend you'd run *because you actually want to use it*.

## Revenue model

Builder fee of 0.05% (5 basis points) attached to every order via
Hyperliquid's on-chain builder code mechanism. Shown transparently
in order previews — honesty is a competitive advantage given
hypefees.com exists and shames wallets that hide theirs.

## Product principles

1. **Full OS, not a skin.** Draggable windows with z-order focus.
   Real taskbar. Real Start menu. Real click-to-minimize behaviors.
2. **No concessions to modernity.** No rounded corners, no smooth
   transitions, no hover animations, no gradients-as-decoration.
   If Win98 didn't have it, hyper98 doesn't either.
3. **Transparency about fees.** The builder fee is shown in every
   order preview. No hiding.
4. **Desktop-only.** Mobile breaks the metaphor. Own that.
5. **Crypto-native only.** External wallets only. No email login,
   no custodial accounts, no hand-holding.

## Scope v1

Markets, Trade, Chart, OrderBook, Positions, Orders, Fills, Wallet,
HIP-3 Markets browser — all as separate draggable windows.

## Non-goals

- Mobile support
- Embedded/custodial wallets
- Fiat onramps
- Copy trading
- Social features
- Aggregation across venues (for now — this is a long-term option)
